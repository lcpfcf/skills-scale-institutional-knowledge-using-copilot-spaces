# OctoAcme — Stakeholder Update Template

## Purpose
Provide a consistent, scannable format for stakeholder communications throughout the project lifecycle. This template is owned and distributed by the **Stakeholder Liaison** in coordination with the **Project Manager** and **Product Manager**.

## When to Use
- Regular cadence updates (weekly, bi-weekly, or milestone-based)
- Post-release announcements
- Escalation or incident notifications

## Usage Notes for the Stakeholder Liaison

- Aim for brevity — stakeholders should be able to scan the update in under 3 minutes.
- Use the status indicator (🟢/🟡/🔴) consistently so recipients can track trend over time.
- Coordinate with the **Project Manager** to ensure risk and milestone data are current before distributing.
- For release-related updates, reference the [Release Readiness Checklist](octoacme-release-readiness-checklist.md) to confirm what has shipped.
- Archive all sent updates in the project's shared drive or repository `/docs` folder for traceability.
